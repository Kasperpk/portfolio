# To prompt or not to prompt

## A new cognitive system

A recent study from The Wharton School by Steven D. Shaw and Gideon Nave titled *Thinking — Fast, Slow and Artificial: How AI is reshaping human reasoning and the rise of cognitive surrender* [1] caught my attention. It did so at the end of day where i had been trying to tackle a particularly challenging data engineering task for my level. Initially i tried my way with it, looking for guides on the internet, documentation of libraries but progress felt slow. Right there to the right in my IDE with a single prompt i had access to my coding assistant and i knew i could produce code without almost any effort on my part. I decided to start the first chat and within 30 seconds i had my notebook filled out with code. This is how it has been like for months, no questions asked, it just produces code as long as i keep giving it a prompt. At the end of that workday i was quite frustrated. First of all the code that was produced introduced new libraries and syntax that i had never seen before. I must also admit almost from first prompt i started to loose the script of what the code did and the reasoning behind each line. The frustration was also that as i ran into errors i had nothing to do but to send it back to the one that produced it. I had become a copy-paster of code and errors.

In Steven D. Shaw and Gideon Nave's paper they investigate what consequences AI can have as people increasingly use it to consult whether it's professionally or personally. A key theory they are testing is that as i have experience myself, excessive reliance on AI creates what they call cognitive surrender. I was no longer involved in thinking and as a consequence my work was no longer feeling satisfying.

For years the theory of human judgement has been understood by the research done by Kahneman and other's in his field [2]. The system 1 and system 2 thinking modes have been used to explain how our judgement can be fast, intuitive and subjective to many biases and heuristics which might not be rational. It can also be slower, more deliberate and analytical, which Kahneman called system 2 thinking. With artificial intelligence, reasoning and judgment can be outsourced to external agents. This is a third cognitive system as Shaw and Nave explains [1]. It's "external, data-driven reasoning originating from algorithmic systems rather than the human mind". This third cognitive reasoning might be used in different ways (Figure 1).

## Why it matters: The consequences of a new tech

When high-level programming slowly replaced assembly language it didn't outsource thinking. Creating software just became more productive because high-level programming had removed unnecessary details such as manipulating memory directly. We might ask ourselves if AI is doing the same. Is it being motivated by removing unnecessary detail? Software engineers might be tired of having to memorize syntax or beating at the keyboard for hours on end. That means what is left might be to understand the real world problem that software might solve. This would then be described to a team of agents that work much like a team of software engineers.

This would require agents to be able to write efficient code that is optimal in terms of time and space complexity. It would also mean it should produce a satisfactory output. So what if no one, no longer codes? How is there then going to be new and better algorithms introduced. How is there going to be new library modules and improvements to how software is written. Such breakthroughs as i see only really comes from working on specific problems for a long time, that which provides rare insights. As we outsource coding and the reasoning behind then the mental model of the program is lost as the code base grows. Some might argue that understanding the code base and is unnecessary detail we don't need to care of while others we see it as system critical, which needs the expertise of a human person.

## Closing thought

I want to bring this post to an end with some closing thoughts. First i think it's about finding balance. Right now many people and organizations are trying to figure these questions out as they go along. Should we hire an senior engineer to oversee a team of agents? Should we retrain our staff to become AI engineers? Should we abandon the technology completely from critical systems or let it loose? I think that with like almost anything in excess, using AI will have negative effects particularly outsourcing deep cognitive work. The paper i have referred to a couple of times now might have some suggestions on how to navigate this new reality [1]. The external judgement might be used as inputs for system 2 thinking, instead of just completely replacing it as it did when i was accepting coding changes without a critical lens. instead of surrendering cognitively and "effectively substituted judgement altogether" i could have asked for inputs to guide me. I could also have described much more detailed what i was building like i was handing over requirements document to a engineer and spent much more time upfront making a plan and iterating over it. The authors present a framework for how to engage this new cognitive system that use "AI not as a passive aid but as a functional cognitive agent whose presence reshapes how and when internal systems engage". We can thereby use the AI to increase our cognitive reasoning by using it in a different way more as a mentor, teacher or conversation partner. Another big takeaway from that article for me is to be aware of when cognitive surrendering versus offloading is happening. Offloading is when we outsource "a discrete task to an external tool", which is more about executing on something well defined.

In a world where there is a constant pull of our attention and the path of least resistance seems so tempting in the moment i think we should remind ourselves of the value craftsmanship. Instead of cognitively surrendering we should use the new tools to enhance our cognitive attention and offload tasks when the hard work has been done.
